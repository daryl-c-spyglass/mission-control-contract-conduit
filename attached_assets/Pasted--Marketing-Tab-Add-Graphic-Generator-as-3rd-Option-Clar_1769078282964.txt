## Graphic Generator - Opens in New Tab (Not iFrame)

Since there's no OAuth setup for the iframe, the Graphic Generator should **open in a new tab** instead of embedding as an iframe. This avoids the login issue.

**When user clicks "Graphic Generator":**

```javascript
// Simply open in new tab - no iframe, no login issues
const openGraphicGenerator = () => {
  const baseUrl = 'https://graphic-1-generator-black--caleb254.replit.app';
  
  // Optionally pass data via URL params (if the tool supports it)
  const params = new URLSearchParams({
    address: transaction.address || '',
    // ... other params if supported
  });
  
  window.open(`${baseUrl}?${params.toString()}`, '_blank');
};
```

## About the iFrame Login Issue

The Graphic Generator at `https://graphic-1-generator-black--caleb254.replit.app` requires authentication, and since it's on a different domain, cookies/sessions don't transfer.

**Options to fix (coordinate with Caleb):**

1. **Open in new tab** (simplest) - User logs in once, stays logged in
2. **Add OAuth/SSO** - Complex, requires Caleb to implement
3. **Public mode** - Caleb adds a `?public=true` mode that skips auth
4. **Token-based** - Pass a temporary token from Contract Conduit

**Recommendation:** For now, just open in new tab. Revisit iframe embedding later if needed.
