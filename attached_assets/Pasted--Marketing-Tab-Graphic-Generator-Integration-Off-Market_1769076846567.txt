## Implementation Notes

### 1. iFrame Communication

```javascript
// Pass data to Graphic Generator via URL params
const iframeUrl = new URL('https://graphic-1-generator-black--caleb254.replit.app');
iframeUrl.searchParams.set('embed', 'true');
iframeUrl.searchParams.set('address', transaction.address);
iframeUrl.searchParams.set('status', getStatus(transaction));
iframeUrl.searchParams.set('agentName', settings.agentName);
iframeUrl.searchParams.set('agentPhone', settings.phoneNumber);
// ... etc

// OR use postMessage after iframe loads
iframe.contentWindow.postMessage({
  type: 'INIT_DATA',
  payload: { address, status, agentName, photos, ... }
}, 'https://graphic-1-generator-black--caleb254.replit.app');
```

### 2. Photo Upload API

```
POST /api/transactions/:id/photos
- Upload new photos (multipart/form-data)

PATCH /api/transactions/:id/photos/:photoId/primary  
- Set as primary photo

DELETE /api/transactions/:id/photos/:photoId
- Delete photo
```

### 3. Graphic Generator Updates Needed

The Graphic Generator app needs to:
- Accept URL params or postMessage for auto-population
- Add `embed=true` mode to hide unnecessary UI
- Send message back to parent when user saves/exports
